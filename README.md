# Required features I have implemented

- API response contains a list/array of tasks.
- Add a single task with title.
- Three diffrent views/tabs such as todos, in-progress, done.
- Store the api response using a global state manager called redux.
- Tasks would be moved 1 step forward or backward via clicking on a specific button/tabs like Move to todo, Move to In progress, Move to Done.
- Task status changes will reflect both on the backend and fronted on each successful response from the backend.
